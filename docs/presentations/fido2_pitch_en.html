What is a FIDO2 security key?

  • Small USB/NFC device that performs cryptographic login with a touch
  • No shared secrets; resistant to phishing, replay, and OTP theft
  • Works on Linux/macOS/Windows for web apps; Linux OS login today
Image: “YubiKey 5 NFC.png”, via Wikimedia Commons (license per source page)

Why FIDO2

  • Stop phishing: cryptographic challenge‑response bound to origin
  • Faster UX: 1‑touch login; no codes, no SMS
  • Lower cost: fewer password resets and lockouts
  • Scales: factory provisioning and CSV logging for up to 500/day

What to expect

  • Defaults: PIN 0000 for new keys, touch‑only login, fingerprint optional
  • Linux login + sudo today (incl. GNOME lock screen); web apps on all platforms
  • Hot‑plug ready: user can plug the key when prompted and just touch

How NADOO‑Key helps companies avoid attacks

  • Eliminates phishable passwords/OTPs for covered systems
  • Touch‑only default avoids fingerprint lockouts and support friction
  • Scripted provisioning + PAM hardening; mapping and permissions automated
  • Factory mode for fast, consistent rollout; CSV for audit and aftercare

How it works (Linux)

  • PAM U2F for login and sudo, with openasuser and hot‑plug helper
  • Per‑user or system‑wide mapping file; correct permissions enforced
  • YubiKey Bio: fingerprint optional; multi‑finger enrollment supported
